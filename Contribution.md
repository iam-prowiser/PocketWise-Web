# Contribution Plan

## Team Collaboration Strategy

PocketWise is being developed collaboratively by four members, with responsibilities distributed according to individual strengths and available resources.

---

# Git Branching Strategy

main
│
├── feature/business-media
├── feature/data-ml
├── feature/frontend
└── feature/backend-integration

* main → Stable production branch.
* Feature branches are used for independent development.
* Changes are merged after review.

# Development Workflow

1. Members work on separate feature branches.
2. Changes are committed regularly.
3. Pull requests are reviewed before merging.
4. Modules are integrated and tested.
5. The main branch always contains the latest stable version.
